# 📌 Trabalho – Testes de Componente

Este projeto foi desenvolvido como parte da atividade de **Testes de Componente**, com o objetivo de validar o comportamento de um sistema de forma integrada e realista.

---

## 🎯 Objetivo

Implementar **testes de componente** que verifiquem o funcionamento do sistema como um todo, integrando múltiplas partes da aplicação.

> ⚠️ Este trabalho **não é**:
>
> * Teste de unidade
> * Uso excessivo de mocks
> * Testes superficiais

Os testes devem refletir o funcionamento real do sistema.

---

## 🛠️ O que foi feito

* Análise do sistema fornecido
* Identificação dos principais fluxos da aplicação
* Implementação de testes de componente utilizando `pytest`
* Criação de cenários de teste incluindo:

  * Casos positivos
  * Casos de erro
* Garantia de execução correta dos testes

---

## 🧠 Abordagem

Os testes foram desenvolvidos buscando:

* Validar o comportamento real do sistema
* Integrar diferentes módulos
* Evitar mocks desnecessários
* Simular situações próximas do uso real

## ▶️ Como executar

1. Clone o repositório:

```bash
git clone <LINK_DO_REPOSITORIO>
```

2. Acesse a pasta:

```bash
cd <NOME_DO_PROJETO>
```

3. Instale as dependências:

```bash
pip install -r requirements.txt
```

4. Execute os testes:

```bash
pytest
```
